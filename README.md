## Important LinkedIn notice

> [!IMPORTANT]
> LinkedIn collection is disabled by default. Public accessibility is not authorization to automate access. Set `INTERNSHIPS_LINKEDIN_CRAWL_AUTHORIZED=true` only after obtaining express permission from LinkedIn. The switch records an operator decision; it does not grant permission.

This independent project is not affiliated with or endorsed by LinkedIn or any listed employer. Listings can change or expire; always verify role details and application requirements on the linked page before applying.

## Overview

The supported pipeline is intentionally narrow:

```text
LinkedIn guest HTML
        ↓
validation and classification
        ↓
SQLite
        ↓
generated README
```

SQLite is the canonical state store. It tracks jobs, searches, run history, provenance, and open/closed lifecycle state. Search-page absence never closes a job; closure requires repeated explicit detail-page unavailability.

Searches are validated YAML files under `configs/searches/`, grouped by role, company, and country. The current registry contains 68 independently bounded searches.

CI validates every change. LinkedIn collection is handled by a separate, controlled, manual GitHub Actions workflow.

See [Architecture](docs/architecture.md) for the complete data flow and component responsibilities.

## Acceptance rules

A listing is published only when every rule passes:

| Check | Required evidence |
|---|---|
| Internship | The title explicitly contains configured internship, placement, or co-op terminology. |
| 2027 cycle | The title or explicit internship-cycle context identifies `2027`; graduation-year eligibility alone is insufficient. |
| Technology role | The title, or a narrowly permitted description fallback, matches a configured technology category. |
| European location | Parsed location text explicitly identifies Europe or a supported European country. |

Senior, staff, principal, lead, manager, director, head-of, and vice-president titles are rejected. Unknown cycle, ambiguous role, and unknown geography are excluded: the project prefers false negatives over false positives.

Rules are configured in [`configs/categories.yml`](configs/categories.yml).

## Quick start

Requirements: Python 3.12 or newer, [`uv`](https://docs.astral.sh/uv/), Git, and express LinkedIn permission only for network collection.

```bash
git clone https://github.com/simonesiega/european-tech-internships-2027.git
cd european-tech-internships-2027
uv sync --frozen --dev
cp .env.example .env
uv run internships db-upgrade
uv run internships searches
uv run internships render
uv run internships validate
```

On Windows Command Prompt, use `copy .env.example .env` instead of `cp`.

Only after receiving express permission, enable the local gate and test one bounded search before running the complete registry:

```dotenv
INTERNSHIPS_LINKEDIN_CRAWL_AUTHORIZED=true
```

```bash
uv run internships search-test company-amazon
uv run internships scrape --search company-amazon
uv run internships validate
```

Docker is optional:

```bash
docker compose build
docker compose run --rm internships --help
```

See [Docker](docs/docker.md) for mounts, permissions, persistence, and authorized collection.

## Essential commands

| Command | Purpose |
|---|---|
| `uv run internships db-upgrade` | Create or upgrade the SQLite schema. |
| `uv run internships searches` | Show enabled searches, effective limits, health, and found/accepted counts. |
| `uv run internships search-test <slug>` | Preview one authorized search without persistence. |
| `uv run internships scrape [--search <slug>]` | Collect, classify, persist, and render all or one authorized search. |
| `uv run internships render` | Regenerate the marked README table from open SQLite jobs. |
| `uv run internships stats` | Show job, search, and run statistics. |
| `uv run internships validate` | Verify migration state, timestamps, and exact SQLite/README agreement. |
| `make check` | Run formatting checks, lint, strict typing, tests, and migration consistency. |

Run `uv run internships --help` for complete CLI help. For common errors and recovery procedures, see [Troubleshooting](docs/troubleshooting.md).

## Configuration summary

The CLI loads built-in defaults, `.env`, an optional settings YAML file, and process environment values in increasing precedence.

The essential settings are:

```dotenv
INTERNSHIPS_DATABASE_URL=sqlite:///data/internships.db
INTERNSHIPS_TARGET_CYCLE=2027
INTERNSHIPS_LINKEDIN_CRAWL_AUTHORIZED=false
```

Never commit `.env` or `configs/settings.yml`. Global search-limit overrides replace evidence-based values from individual search files.

See [Configuration](docs/configuration.md) for every `INTERNSHIPS_*` variable, default, validation rule, and precedence detail.
